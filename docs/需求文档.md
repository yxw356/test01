# 企业知识中枢 — 需求文档

> 版本：v1.1  
> 更新日期：2026-05-20  
> 说明：本文档描述系统功能需求、非功能需求、部署约束与验收标准，与当前代码实现保持一致。

---

## 1. 项目目标

建设面向企业的**私有知识库 RAG 智能问答平台**，实现：

- 企业知识文件（制度、流程、项目资料、方案、培训材料等）的统一上传与管理；
- 按用户、组织标签、公开范围进行**权限隔离**的检索与问答；
- 基于**混合检索（BM25 + 向量）+ RAG** 的可追溯问答，答案须来源于企业知识库内容；
- 支持多轮对话、流式输出与管理端运维能力。

## 2. 用户角色

| 角色 | 说明 | 主要能力 |
| --- | --- | --- |
| 普通用户（USER） | 企业员工 | 上传/管理本人知识文件、混合检索、智能问答、个人中心 |
| 管理员（ADMIN） | 系统管理员 | 普通用户全部能力 + 用户管理、组织标签、聊天记录查看 |

默认管理员：`admin` / `admin123`（首次启动自动初始化）。

## 3. 功能需求

### 3.1 认证与权限

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-AUTH-01 | 用户名密码登录，签发 JWT，支持 Token 刷新与注销 | P0 |
| FR-AUTH-02 | 接口按角色鉴权；未登录用户不可访问业务 API | P0 |
| FR-AUTH-03 | 知识检索与问答仅返回当前用户**有权限**的文档块（本人 / 公开 / 所属组织及层级标签） | P0 |
| FR-AUTH-04 | 组织标签支持层级关系；用户有效标签含默认组织 `DEFAULT` | P1 |

### 3.2 知识库文件管理

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-KB-01 | 支持大文件**分片上传**、合并、断点续传、上传状态查询 | P0 |
| FR-KB-02 | 文件元数据写入 MySQL；合并后对象存储至 **MinIO**（桶名 `uploads`） | P0 |
| FR-KB-03 | 合并成功后通过 **Kafka** 异步触发：下载 → Tika 解析 → 切分 → 向量化 → 写入 ES | P0 |
| FR-KB-04 | 支持 PDF / Word / TXT 等常见格式（Apache Tika） | P0 |
| FR-KB-05 | 父子块切分：子块用于检索，父块用于聚合展示上下文 | P1 |
| FR-KB-06 | 文件列表按权限过滤；支持删除（MinIO + MySQL + ES 联动清理） | P1 |
| FR-KB-07 | 上传时可设置**是否公开**、**组织标签**（如 `default`） | P0 |

### 3.3 检索

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-SEARCH-01 | 提供混合检索 API：`GET /api/v1/search/hybrid?query=&topK=` | P0 |
| FR-SEARCH-02 | 单路召回包含：原始 query 的 BM25 + 向量 KNN，RRF 融合后按父块聚合 | P0 |
| FR-SEARCH-03 | 知识库弹窗内可测试混合检索（前端代理 `/proxy-api`） | P1 |
| FR-SEARCH-04 | 向量维度与 Embedding 模型、ES `dense_vector.dims` **必须一致**（当前为 **1024 / bge-m3**） | P0 |

### 3.4 智能问答（RAG）

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-CHAT-01 | WebSocket 流式问答；基于检索上下文 + 系统 Prompt 规则生成回答 | P0 |
| FR-CHAT-02 | 多轮对话：Redis 维护 `conversationId` 与近期历史 | P0 |
| FR-CHAT-03 | RAG 流水线：Query 理解 → 多路召回 → 候选过滤（RRF / Cross-Encoder 占位 / MMR）→ 可选 Reflection → 最终 Top-K | P1 |
| FR-CHAT-04 | Query 改写、HyDE、Reflection 可通过配置开关；**本地大模型环境建议关闭**以降低延迟 | P1 |
| FR-CHAT-05 | 无检索结果时，模型应说明「暂无相关信息」，不得编造企业事实 | P0 |

### 3.5 管理功能

| 编号 | 需求 | 优先级 |
| --- | --- | --- |
| FR-ADMIN-01 | 用户管理：列表、创建、禁用、组织标签分配 | P1 |
| FR-ADMIN-02 | 组织标签管理 | P1 |
| FR-ADMIN-03 | 聊天记录查询（管理端） | P2 |

## 4. 非功能需求

| 编号 | 类别 | 需求 |
| --- | --- | --- |
| NFR-01 | 安全 | 内部资料不出域；JWT 密钥可配置；敏感配置不入库 |
| NFR-02 | 可用性 | 上传与索引异步解耦；索引失败可在日志中定位并重试（Kafka 重试 / 重启后 ES 维度修复自动重建） |
| NFR-03 | 性能 | 混合检索（关闭 Query 改写/HyDE）目标在秒级；完整 RAG Stage1 开启 LLM 增强时允许分钟级（受本地 GPU 约束） |
| NFR-04 | 可维护性 | ES 索引启动时校验向量维度；与 `embedding.api.dimension` 不一致时自动删索引重建并从 MySQL 分块重灌 |
| NFR-05 | 兼容性 | LLM / Embedding 均通过 **OpenAI 兼容 HTTP API** 接入（Ollama、vLLM 等） |

## 5. 外部依赖与接口

### 5.1 基础设施（Docker Compose）

| 组件 | 默认端口 | 用途 |
| --- | --- | --- |
| MySQL | 3307 | 用户、文件元数据、分块文本 |
| Redis | 6379 | 会话、Token 缓存 |
| MinIO | 19000 / 19001 | 对象存储；**须存在桶 `uploads`**（应用启动可自动创建） |
| Kafka | 9092 | 文件处理任务队列 |
| Elasticsearch | 9200 | 知识索引 `knowledge_base` |

### 5.2 模型服务（推荐本地 vLLM 双实例）

| 能力 | 默认配置 | 说明 |
| --- | --- | --- |
| 对话 LLM | `http://127.0.0.1:8000/v1`，模型如 `Qwen3.6-35B-A3B` | `/chat/completions` |
| 向量 Embedding | `http://127.0.0.1:8001/v1`，模型 `bge-m3` | **独立进程**；对话实例通常不提供 `/embeddings` |
| 向量维度 | `embedding.api.dimension: 1024` | 与 ES mapping `vector.dims` 一致 |

配置项见 `src/main/resources/application.yml`：`deepseek.api.*`、`embedding.api.*`、`rag.*`。

### 5.3 应用服务端口

| 服务 | 端口 |
| --- | --- |
| 后端 API | 8081 |
| 前端开发服 | 9527 |

## 6. 数据与索引规范

### 6.1 Elasticsearch 索引 `knowledge_base`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| fileMd5 | keyword | 文件指纹 |
| chunkId | integer | 分块序号 |
| parentId | keyword | 父块 ID |
| textContent | text (ik) | 检索正文 |
| parentTextContent | text | 父块全文（可不索引） |
| vector | dense_vector, **dims=1024** | 与 bge-m3 输出一致 |
| userId | keyword | 上传用户 DB ID |
| orgTag | keyword | 组织标签（注意与 `DEFAULT` 大小写兼容） |
| isPublic | boolean | 是否公开 |

### 6.2 索引写入成功条件

1. MinIO 桶 `uploads` 存在且分片合并成功；  
2. Kafka 消费完成解析，分块写入 `document_vectors`；  
3. Embedding API 返回 1024 维向量；  
4. ES bulk 无维度/映射错误。

**常见失败**：映射仍为 768 维、Embedding 服务未启动、MinIO `NoSuchBucket`。

## 7. 配置说明（RAG 与性能）

`application.yml` 中 `rag.pipeline` 建议：

| 配置项 | 本地开发建议 | 生产/高配 GPU 可选 |
| --- | --- | --- |
| enable-query-rewrite | `false` | `true` |
| enable-hyde | `false` | `true` |
| enable-reflection | `false` | `true` |

关闭上述开关后，问答检索主要依赖原始 query 的 BM25 + 向量双路召回，延迟显著降低。

## 8. 验收标准

| 场景 | 验收条件 |
| --- | --- |
| 文件入库 | 上传 txt/pdf 后，Kafka 日志无持续报错；`curl ES/knowledge_base/_count` 大于 0 |
| 混合检索 | 登录后检索文档内关键词（如企业专有名词），返回带 `textContent` 的命中列表 |
| 智能问答 | 同一关键词提问，回答引用知识库内容或明确「暂无相关信息」 |
| 权限 | 非公开、非本组织文件不对无权限用户召回 |
| 健康检查 | `GET http://localhost:8081/` 返回 JSON 健康信息 |

## 9. 运维与排障

| 现象 | 可能原因 | 处理建议 |
| --- | --- | --- |
| 聊天搜不到已上传内容 | ES 文档数为 0 或检索反序列化失败 | 查 `knowledge_base/_count`、后端日志 `bulkIndex` / `Failed to decode`；确认维度 1024、重启后端触发重建 |
| 上传 500 MinIO | 桶不存在 | 确认 `uploads` 桶；或依赖 `MinioBucketInitializer` |
| 问答极慢 | Query 改写/HyDE 多次调用大模型 | 将 `rag.pipeline` 三项开关设为 `false` |
| 向量 404 | 仅启动对话 vLLM | 另起 8001 embedding 服务（bge-m3） |

应急脚本（可选）：`scripts/reindex-from-mysql.py` — 在 ES 映射已正确时，从 MySQL 分块重建索引。

完整安装步骤见 [操作手册.md](./操作手册.md)。
