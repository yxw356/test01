## 1. 你将搭建什么

系统由 **5 层** 组成，启动顺序很重要：

```
浏览器 (9527)
    ↓
Spring Boot 后端 (8081) ←→ MySQL / Redis / MinIO / Kafka / Elasticsearch
    ↓
vLLM 对话 (8000) + vLLM 向量 (8001)
```

**一条知识文件的生命周期：**

1. 前端分片上传 → MinIO 存储  
2. Kafka 异步任务 → Tika 解析 → 文本切分 → 写入 MySQL `document_vectors`  
3. 调用 Embedding API 生成向量 → 写入 Elasticsearch `knowledge_base`  
4. 用户提问 → 混合检索（BM25 + 向量）→ 拼上下文 → LLM 流式回答  

**新手必须记住的三件事：**

| 要点 | 说明 |
| --- | --- |
| 向量维度要一致 | bge-m3 为 **1024 维**，ES 索引 `vector.dims` 也必须是 **1024** |
| 对话和向量分开 | 大模型端口（8000）通常**没有** `/embeddings`，向量要单独起 **8001** |
| 上传≠立即可搜 | 需等 Kafka 消费完成，ES `_count` > 0 后再问答 |

## 2. 环境与硬件建议

### 2.1 操作系统

- 推荐：**Ubuntu 22.04+** 或同类 Linux（本手册以 Linux 为例）
- 也可用 macOS；Windows 建议 WSL2 + Docker Desktop

### 2.2 硬件（本地完整体验）

| 组件 | 最低建议 | 说明 |
| --- | --- | --- |
| CPU | 8 核+ | 跑 Docker 中间件 |
| 内存 | 32 GB+ | ES 单独约占 2 GB；大模型另占大量内存 |
| 磁盘 | 100 GB+ 可用 | 模型权重 + Docker 卷 |
| GPU | 可选但强烈建议 | 对话模型（35B 级）与 bge-m3 向量；无 GPU 可改用云端 API 或小模型 |

### 2.3 软件版本

| 软件 | 版本 |
| --- | --- |
| Docker | 24+ |
| Docker Compose | v2 |
| JDK | **17**（不要用 8/11） |
| Maven | 3.9+ |
| Node.js | 18.20+ |
| pnpm | 8.7+ |

## 4. 第一步：安装基础工具

### 4.1 Docker

按官方文档安装 Docker Engine 与 Compose 插件。安装后：

```bash
sudo usermod -aG docker $USER
# 重新登录 shell 后
docker ps
```

### 4.2 JDK 17

**方式 A（推荐）：** 下载 Temurin 17 解压到用户目录，例如：

```bash
mkdir -p ~/.local/tools
# 将 JDK 解压到 ~/.local/tools/jdk-17.0.x
export JAVA_HOME=$HOME/.local/tools/jdk-17
export PATH=$JAVA_HOME/bin:$PATH
```

写入 `~/.bashrc` 以便永久生效。

**方式 B：** 系统包管理器安装 `openjdk-17-jdk`。

验证：`java -version` 输出含 `17`。

### 4.3 Maven

```bash
# 示例：SDKMAN
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk install maven 3.9.9
```

### 4.4 Node.js 与 pnpm

```bash
# 示例：Node 18 + 全局 pnpm
npm install -g pnpm@8.15.9
```

## 5. 第二步：启动 Docker 基础设施

### 5.1 准备目录（可选）

`docs/docker-compose.yaml` 中部分卷映射到 `/data/docker/...`。若本机无此路径，可先创建空目录，避免权限告警：

```bash
sudo mkdir -p /data/docker/mysql/conf /data/docker/minio/config /data/docker/redis
sudo chown -R $USER:$USER /data/docker
```

若不想使用主机路径，可编辑 compose 文件删除对应 `volumes` 下的主机绑定行，仅保留命名卷。

### 5.2 拉取镜像（国内网络）

若 `docker pull` 超时，可对**单个镜像**使用镜像加速前缀，例如：

```bash
docker pull docker.m.daocloud.io/library/mysql:8
docker tag docker.m.daocloud.io/library/mysql:8 mysql:8
# 对其余镜像（redis、minio、elasticsearch、bitnamilegacy/kafka）同理
```

### 5.3 启动全部中间件

在项目根目录执行：

```bash
cd /path/to/test01
docker compose -f docs/docker-compose.yaml up -d
```

等待约 1～2 分钟，检查状态：

```bash
docker compose -f docs/docker-compose.yaml ps
```

期望看到 `mysql`、`redis`、`minio`、`kafka`、`es` 均为 running。Kafka 首次启动会创建主题 `file-processing`，需稍等。

### 5.4 验证各中间件

| 服务 | 验证命令 | 期望 |
| --- | --- | --- |
| MySQL | `docker exec mysql mysql -uroot -pPaiSmart2025 -e "SELECT 1"` | 无报错 |
| Redis | `docker exec redis redis-cli -a PaiSmart2025 ping` | `PONG` |
| MinIO | 浏览器打开 `http://localhost:19001` | 控制台可登录 `admin` / `PaiSmart2025` |
| ES | `curl -u elastic:PaiSmart2025 http://localhost:9200` | 返回集群 JSON |
| Kafka | `docker exec kafka kafka-topics.sh --bootstrap-server localhost:9092 --list` | 含 `file-processing` |

### 5.5 MySQL 数据库

后端使用库名 **`PaiSmart`**（见 `application.yml`）。首次启动后端时 JPA `ddl-auto: update` 会自动建表，一般**无需**手动导入 DDL。若需手工建库：

```bash
docker exec mysql mysql -uroot -pPaiSmart2025 -e "CREATE DATABASE IF NOT EXISTS PaiSmart DEFAULT CHARSET utf8mb4;"
```

## 6. 第三步：配置并启动 AI 模型服务

### 6.1 为什么需要两个端口？

| 端口 | 用途 | API |
| --- | --- | --- |
| **8000** | 对话大模型 | `POST /v1/chat/completions` |
| **8001** | 向量模型 bge-m3 | `POST /v1/embeddings` |

**经验：** 只起 8000 时，上传文件后向量化会失败或返回 404，ES 里永远是 0 条文档。

### 6.2 启动对话模型（8000）

按你本机已有方式启动 vLLM，例如：

```bash
# 示例，路径与模型名请按实际修改
vllm serve /path/to/Qwen3.6-35B-A3B \
  --host 127.0.0.1 --port 8000 \
  --api-key vllm_api_key_12345
```

验证：

```bash
curl -s -H "Authorization: Bearer vllm_api_key_12345" \
  http://127.0.0.1:8000/v1/models
```

### 6.3 启动向量模型（8001）

bge-m3 占用显存较小，可与对话模型**同卡**共存（需调低 `--gpu-memory-utilization`）：

```bash
vllm serve /path/to/bge-m3 \
  --runner pooling \
  --convert embed \
  --host 127.0.0.1 \
  --port 8001 \
  --api-key vllm_api_key_12345 \
  --served-model-name bge-m3 \
  --gpu-memory-utilization 0.04 \
  --max-model-len 2048
```

验证：

```bash
curl -s -H "Authorization: Bearer vllm_api_key_12345" \
  -H "Content-Type: application/json" \
  http://127.0.0.1:8001/v1/embeddings \
  -d '{"model":"bge-m3","input":["测试"],"dimension":1024}'
```

应返回 `data[0].embedding` 且长度约为 **1024**。

### 6.4 无 GPU 时的替代方案

- 使用 **Ollama** 同时提供 chat 与 embed（需选定向量维度与 ES 一致的模型，并改 `application.yml`）  
- 或使用云端 OpenAI 兼容 API，填写 `deepseek.api.url` / `embedding.api.url` 与 key  

无论哪种方式，**`embedding.api.dimension` 必须与 ES 映射 `dims` 一致**。

## 7. 第四步：配置应用并启动后端

### 7.1 核对 `application.yml`

文件路径：`src/main/resources/application.yml`

**关键配置项：**

```yaml
# 中间件（与 docker-compose 默认一致）
spring.datasource.url: jdbc:mysql://localhost:3307/PaiSmart?...
spring.data.redis.password: PaiSmart2025
minio.endpoint: http://localhost:19000
minio.bucketName: uploads
elasticsearch.host: localhost
elasticsearch.password: PaiSmart2025

# 对话模型
deepseek.api.url: http://127.0.0.1:8000/v1
deepseek.api.model: <你的对话模型名>
deepseek.api.key: vllm_api_key_12345

# 向量模型（独立端口）
embedding.api.url: http://127.0.0.1:8001/v1
embedding.api.model: bge-m3
embedding.api.dimension: 1024    # 必须与 ES 一致
embedding.api.key: vllm_api_key_12345

# 本地开发建议（避免问答前等待数分钟）
rag.pipeline.enable-query-rewrite: false
rag.pipeline.enable-hyde: false
rag.pipeline.enable-reflection: false
```

### 7.2 启动后端

```bash
cd /path/to/test01
export JAVA_HOME=$HOME/.local/tools/jdk-17   # 按你的路径
export PATH=$JAVA_HOME/bin:$PATH
mvn spring-boot:run
```

首次启动日志中应看到：

- `索引 'knowledge_base' 已创建` 或 `向量维度=1024`  
- `MinIO 存储桶 'uploads' 已存在` 或已创建  
- `Tomcat started on port 8081`  

健康检查：

```bash
curl http://127.0.0.1:8081/
```

返回 JSON 且含 `"status":"ok"` 即正常。

> **说明：** 后端是纯 API，浏览器直接打开 8081 没有完整页面；请使用前端 9527。

### 7.3 端口被占用

若提示 `Address already in use`：

```bash
ss -tlnp | grep 8081
kill <pid>
```

再重新 `mvn spring-boot:run`。

## 9. 第六步：验收测试

按顺序做，全部通过再交给业务试用。

### 9.1 登录

打开 `http://localhost:9527/#/login`，使用 `admin` / `admin123` 登录成功。

### 9.2 上传知识文件

1. 进入 **知识库**  
2. 上传一个 **TXT 或 PDF**（内容尽量有明确关键词，如公司介绍）  
3. 等待上传完成（进度 100%）  

### 9.3 确认索引已写入 ES

```bash
curl -s -u elastic:PaiSmart2025 http://localhost:9200/knowledge_base/_count
```

期望：`"count":` **大于 0**。若为 0，见 [11. 常见问题](#11-常见问题与排障)。

也可查看后端日志是否出现：

- `文件解析完成`  
- `向量化完成`  
- `批量索引成功完成`  

### 9.4 混合检索

知识库内打开 **混合检索**，输入文件中出现过的一两个关键词，应返回命中片段。

### 9.5 智能问答

进入 **智能问答**，输入同一关键词提问，回答应引用知识库内容，或明确说明「暂无相关信息」（不应凭空编造）。

## 10. 日常使用流程

| 步骤 | 操作 |
| --- | --- |
| 1 | 登录系统 |
| 2 | 知识库 → 上传文件 → 等待处理完成 |
| 3 | （可选）混合检索验证是否入库 |
| 4 | 智能问答中提问 |
| 5 | 管理员可在用户管理 / 组织标签 / 聊天记录中运维 |

**上传后多久可搜？** 通常数十秒到数分钟，取决于文件大小与 GPU 向量服务负载。大 PDF 更久。

**文件权限：** 上传时可设「公开」与组织标签；仅本人、公开文档、或有权限的组织文档会被检索到。

## 11. 常见问题与排障

### 11.1 上传失败：MinIO `NoSuchBucket`

| 项目 | 内容 |
| --- | --- |
| 现象 | 分片上传 500，`The specified bucket does not exist` |
| 原因 | MinIO 中无 `uploads` 桶 |
| 处理 | 重启后端（会自动创建）；或 MinIO 控制台手动建桶 `uploads`；或 `mc mb local/uploads` |

### 11.2 已上传但检索/问答无内容

| 项目 | 内容 |
| --- | --- |
| 现象 | 知识库有文件，问关键词无结果 |
| 排查 1 | `curl -u elastic:PaiSmart2025 http://localhost:9200/knowledge_base/_count` 是否为 0 |
| 排查 2 | 日志搜 `bulkIndex`、`more dimensions than defined in the mapping` |
| 原因 A | ES 映射 768 维，bge-m3 输出 1024 维 → **索引写入失败** |
| 原因 B | 8001 向量服务未启动 |
| 原因 C | 检索反序列化失败（历史问题，新版本已修复 `isPublic` 与 vector 字段） |
| 处理 | 确认 `embedding.api.dimension: 1024`；重启后端触发索引维度校验与 MySQL 重灌；确认 8001 正常 |

### 11.3 Embedding 接口 404

对话 vLLM（8000）不提供 `/embeddings`。必须单独启动 8001 的 bge-m3 服务。

### 11.4 问答非常慢（数分钟才有回复）

| 项目 | 内容 |
| --- | --- |
| 原因 | `rag.pipeline` 开启 query-rewrite、HyDE、Reflection，Stage1 多次调用 35B 模型 |
| 处理 | 三项均设为 `false`（见 7.1）；生产高配环境再按需开启 |

### 11.5 混合检索弹窗有结果，聊天仍无结果

确认 ES `_count` > 0；重启后端后再试；聊天使用用户名 `admin` 做权限，与上传用户一致或文件设为公开。

### 11.6 Docker 拉镜像失败

使用国内镜像站逐镜像拉取并 `docker tag` 为 compose 中名称（见 5.2）。

### 11.7 8081 无法访问 / 空白

后端无首页 UI，请访问 **9527** 前端；API 用 `curl http://127.0.0.1:8081/` 测健康接口。

### 11.8 GPU OOM 启动 bge-m3 失败

降低 `--gpu-memory-utilization`（如 `0.04`）；或先停对话模型再启向量；或使用 CPU/Ollama 小向量模型。

### 11.9 查看日志

| 日志 | 路径 |
| --- | --- |
| 业务/检索 | `logs/business.*.log` |
| 错误 | `logs/error.*.log` |
| 后端控制台 | `mvn spring-boot:run` 终端输出 |

## 12. 本次部署经验总结

以下为实际联调中踩坑与结论，写入手册便于团队复用。

### 12.1 架构层面

1. **异步索引链路长**：上传成功 ≠ 可检索，必须验证 ES。  
2. **RAG 分阶段耗时差异大**：关闭 LLM 增强后检索可在秒级；全开可能 100+ 秒。  
3. **权限在 ES 过滤层实现**：`userId`、`isPublic`、`orgTag` 写入索引时必须正确。

### 12.2 配置层面

1. **向量三维对齐**：`embedding.api.dimension` = ES `dense_vector.dims` = 模型输出维度（bge-m3 → 1024）。  
2. **双 vLLM 进程**：8000 对话 + 8001 向量，不可省。  
3. **MinIO 桶名** `uploads` 与配置一致，建议启动时自动创建。  
4. **组织标签大小写**：库中可能为 `default`，权限缓存为 `DEFAULT`，检索 filter 需兼容。

### 12.3 代码与运维改进（已合入项目）

| 改进 | 作用 |
| --- | --- |
| ES 映射改为 1024 维 | 匹配 bge-m3 |
| 启动时校验 ES 维度并重建索引 | 避免旧 768 映射残留 |
| 从 MySQL 分块自动重灌 ES | 修复后无需重新上传 |
| 检索排除 `vector` 字段 + `EsSearchDocument` | 避免大向量 JSON 反序列化失败 |
| `isPublic` 使用 `Boolean` + `@JsonProperty` | 避免 Jackson 映射为 `public` 字段 |
| `rag.pipeline` 开关真正生效 | 关闭后跳过 Query 改写/HyDE |
| `MinioBucketInitializer` | 自动创建 uploads 桶 |
| `GET /` 健康接口 | 便于确认后端存活 |

### 12.4 推荐生产检查表

- [ ] Docker 五件套健康  
- [ ] 8000 / 8001 模型 API 可用  
- [ ] `application.yml` 密码与端口正确  
- [ ] ES `_count` 在试上传后 > 0  
- [ ] 混合检索 + 问答各测一条  
- [ ] `rag.pipeline` 按机器性能设置开关
