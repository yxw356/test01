# 企业能力落地说明

> 审计日志、权限策略、表格解析、运行监控 — 面向企业内部长期使用。

---

## 1. 审计日志

### 数据表

启动后端后 JPA 自动创建 `audit_log` 表（或执行 `docs/databases/ddl.sql`）。

### 记录范围

| 操作 | action | 触发点 |
| --- | --- | --- |
| 登录 | LOGIN | 用户登录 |
| 上传 | UPLOAD | 分片合并成功 |
| 删除 | DELETE | 文档删除 |
| 检索 | SEARCH | 混合检索 |
| 预览 | PREVIEW | 文档预览 |
| 下载 | DOWNLOAD | 文档下载 |
| 问答 | CHAT | WebSocket 提问 |
| 索引成功 | INDEX_SUCCESS | Kafka 消费完成 |
| 索引失败 | INDEX_FAILURE | Kafka 消费异常 |

### 管理员查询

前端菜单：**审计日志**、**运行监控**（需 ADMIN 角色）。

```bash
curl -H "Authorization: Bearer <ADMIN_TOKEN>" \
  "http://127.0.0.1:8081/api/v1/admin/audit-logs?page=0&size=20&action=SEARCH"
```

### 索引失败重试

知识库列表中，索引状态为「待索引 / 索引中 / 索引失败」时可点击 **重试索引**，或调用：

```bash
curl -X POST -H "Authorization: Bearer <TOKEN>" \
  http://127.0.0.1:8081/api/v1/documents/<FILE_MD5>/reindex
```

## 2. 权限隔离

- ES 检索层：`HybridSearchService.buildPermissionFilter`
- 二次校验：`DocumentAccessPolicy.canAccess`（返回结果再过滤）
- 自动化测试：`DocumentAccessPolicyTest`（CI 必跑）

验收场景：

1. orgA 用户上传私有文档 → orgB 用户检索 0 命中  
2. 公开文档 → 任意登录用户可检索  
3. `default` / `DEFAULT` 组织标签大小写兼容  

运行测试：

```bash
mvn test -Dtest=DocumentAccessPolicyTest
```

## 3. 表格解析

Excel（`.xlsx`/`.xls`）与 CSV 走 `TabularParseService`，输出结构化块：

```text
[TABLE source=Excel sheet=Sheet1 columns=区域|季度|金额]
区域=华东 | 季度=Q3 | 金额=1200
[/TABLE]
```

便于「Q3 华东销售额是多少」类数值问答。表格型 PDF 仍走 Tika，后续可接 tabula-java。

测试：

```bash
mvn test -Dtest=TabularParseServiceTest
```

## 4. 运行监控

### Actuator

- 健康检查：`GET http://127.0.0.1:8081/actuator/health`（匿名可访问）
- 指标：`GET http://127.0.0.1:8081/actuator/metrics`

### 管理员聚合面板

```bash
curl -H "Authorization: Bearer <ADMIN_TOKEN>" \
  http://127.0.0.1:8081/api/v1/admin/monitoring/status
```

返回示例字段：

| 字段 | 说明 |
| --- | --- |
| components.redis | Redis 状态 |
| components.elasticsearch.knowledgeBaseCount | ES 文档数 |
| components.vllmChat / vllmEmbedding | 8000 / 8001 模型服务 |
| components.kafka.totalLag | Kafka 消费积压 |
| metrics.indexSuccessCount / indexFailureCount | 索引成功/失败计数 |
| metrics.chatAverageDurationMs | 问答平均耗时 |

### 建议告警阈值

- vLLM DOWN → 立即告警  
- Kafka totalLag > 100 持续 5min → 告警  
- indexFailureCount 1h 内 > 3 → 告警  
- chatP95EstimateMs > 30000 → 告警
