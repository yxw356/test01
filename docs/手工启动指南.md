# 手工启动指南

> 面向 **test01（企业知识中枢）** 的日常手工启动速查。  
> 首次安装、环境准备与排障详见 [操作手册](./操作手册.md)。

---

## 启动顺序

```
Docker 中间件
    ↓
vLLM 对话 (8000)
    ↓
vLLM 向量 (8001)
    ↓
Spring Boot 后端 (8081)
    ↓
Vue 前端 (9527)
```

**项目路径：** `/home/lhagent/test01`（请按本机实际路径调整）

---

## 第 1 步：Docker 中间件

```bash
cd /home/lhagent/test01
docker compose -f docs/docker-compose.yaml up -d
```

检查状态（期望 mysql、redis、minio、kafka、es 均为 running）：

```bash
docker compose -f docs/docker-compose.yaml ps
```

| 服务 | 端口 | 默认账号 |
| --- | --- | --- |
| MySQL | 3307 | root / PaiSmart2025 |
| Redis | 6379 | 密码 PaiSmart2025 |
| MinIO API | 19000 | admin / PaiSmart2025 |
| MinIO 控制台 | 19001 | admin / PaiSmart2025 |
| Kafka | 9092 | — |
| Elasticsearch | 9200 | elastic / PaiSmart2025 |

---

## 第 2 步：对话模型（终端 A，端口 8000）

单独开一个终端，保持运行：

```bash
vllm serve Qwen3.6-35B-A3B \
  --host 127.0.0.1 --port 8000 \
  --api-key vllm_api_key_12345
```

验证：

```bash
curl -s -H "Authorization: Bearer vllm_api_key_12345" \
  http://127.0.0.1:8000/v1/models
```

> 模型名须与 `src/main/resources/application.yml` 中 `deepseek.api.model` 一致（当前为 `Qwen3.6-35B-A3B`）。

---

## 第 3 步：向量模型（终端 B，端口 8001）

**必须单独启动。** 8000 端口不提供 `/embeddings`，否则上传文件后无法向量化，ES 中永远为 0 条文档。

```bash
vllm serve BAAI/bge-m3 \
  --runner pooling --convert embed \
  --host 127.0.0.1 --port 8001 \
  --api-key vllm_api_key_12345 \
  --served-model-name bge-m3 \
  --gpu-memory-utilization 0.04 \
  --max-model-len 2048
```

验证（返回向量长度约 **1024**）：

```bash
curl -s -H "Authorization: Bearer vllm_api_key_12345" \
  -H "Content-Type: application/json" \
  http://127.0.0.1:8001/v1/embeddings \
  -d '{"model":"bge-m3","input":["测试"],"dimension":1024}'
```

> 向量维度须与 `embedding.api.dimension: 1024` 及 ES 索引一致。

---

## 第 4 步：后端（终端 C，端口 8081）

```bash
cd /home/lhagent/test01
export JAVA_HOME=$HOME/.local/tools/jdk-17   # 按本机 JDK 17 路径调整
export PATH=$JAVA_HOME/bin:$PATH
mvn spring-boot:run
```

成功标志：日志出现 `Tomcat started on port 8081`。

健康检查：

```bash
curl http://127.0.0.1:8081/
```

期望返回 JSON 且含 `"status":"ok"`。

> 后端为纯 API，浏览器直接打开 8081 无完整页面，请使用前端 9527。

---

## 第 5 步：前端（终端 D，端口 9527）

```bash
cd /home/lhagent/test01/frontend
pnpm install    # 首次或依赖变更时需要
pnpm dev
```

浏览器访问：**http://localhost:9527**

- 登录账号：`admin` / `admin123`
- 远程访问时使用服务器 IP，例如 `http://192.168.x.x:9527`，勿用 `localhost`

## 日常快速启动

若 Docker 中间件已在运行（重启机器后需重新执行第 1 步），通常只需开 **4 个终端**：

1. vLLM 对话（8000）
2. vLLM 向量（8001）
3. `mvn spring-boot:run`
4. `pnpm dev`

## 验收自检

```bash
# 后端存活
curl http://127.0.0.1:8081/

# ES 是否有索引文档（上传文件并等待 Kafka 消费后）
curl -s -u elastic:PaiSmart2025 http://localhost:9200/knowledge_base/_count
```

上传文件后 `count` 应大于 0，方可进行混合检索与智能问答。

---

## 常见问题

| 现象 | 处理 |
| --- | --- |
| 上传成功但检索/问答无结果 | 确认 8001 已启动；检查 ES `_count` 是否为 0 |
| Embedding 404 | 对话 vLLM（8000）无 `/embeddings`，必须起 8001 |
| 8081 端口被占用 | `ss -tlnp \| grep 8081` 后 kill 对应进程再重启 |
| 问答极慢 | `application.yml` 中 `rag.pipeline` 三项建议保持 `false` |
| MinIO NoSuchBucket | 重启后端（会自动创建 `uploads` 桶）或 MinIO 控制台手动建桶 |

更多排障见 [操作手册 · 常见问题](./操作手册.md#11-常见问题与排障)。
